# Sample Data Upload

Upload local CSV files to a Fabric Lakehouse's `Files/` section using the OneLake DFS API (ADLS Gen2-compatible).

## Prerequisites

- A valid Fabric Bearer token stored in `$FABRIC_TOKEN`
- `infra/workspace_ids.json` generated by running [deploy_medallion_workspaces.py](../scripts/deploy_medallion_workspaces.py) at least once

## Quick Start

Upload `data/sample_raw_sales.csv` to the **dev** workspace, **bronze** lakehouse, at `Files/raw/sample_raw_sales.csv`:

```bash
python scripts/upload_sample_data.py --token "$FABRIC_TOKEN"
```

This reads target IDs automatically from `infra/workspace_ids.json`.

## All Options

| Flag | Default | Description |
|---|---|---|
| `--token` | *(required)* | Fabric Bearer access token |
| `--env` | `dev` | Target environment (`dev`, `prod`, `feature`, `staging`) |
| `--layer` | `bronze` | Target medallion lakehouse layer (`raw`, `bronze`, `silver`, `gold`) |
| `--file` | `data/sample_raw_sales.csv` | Local file to upload |
| `--remote-path` | `raw/sample_raw_sales.csv` | Destination path inside the lakehouse `Files/` root |
| `--workspace-ids` | `infra/workspace_ids.json` | Path to the workspace IDs file |
| `--workspace-id` | — | Override: Fabric workspace GUID (bypasses workspace_ids.json lookup) |
| `--lakehouse-id` | — | Override: Fabric lakehouse GUID (bypasses workspace_ids.json lookup) |

## Examples

**Upload to the raw lakehouse instead of bronze:**

```bash
python scripts/upload_sample_data.py --token "$FABRIC_TOKEN" --layer raw
```

**Target a different environment:**

```bash
python scripts/upload_sample_data.py --token "$FABRIC_TOKEN" --env staging
```

**Custom local file and destination path:**

```bash
python scripts/upload_sample_data.py --token "$FABRIC_TOKEN" \
    --file data/my_data.csv \
    --remote-path landing/my_data.csv
```

**Skip workspace_ids.json and supply IDs directly:**

```bash
python scripts/upload_sample_data.py --token "$FABRIC_TOKEN" \
    --workspace-id bafe69f9-6a07-4f62-a1bc-de1fef9e9086 \
    --lakehouse-id 468c2a4f-43de-458c-97da-0f4f71fe3480
```

## How It Works

The script uses the OneLake DFS endpoint (`https://onelake.dfs.fabric.microsoft.com`) which implements the ADLS Gen2 protocol. The upload happens in three API calls:

1. **Create** — `PUT /workspaceId/lakehouseId/Files/<path>?resource=file` — creates an empty file entry
2. **Append** — `PATCH ...?action=append&position=0` — writes the file bytes
3. **Flush** — `PATCH ...?action=flush&position=<size>` — commits the data

The same Entra Bearer token used by `deploy_medallion_workspaces.py` works here.

## Where the File Lands in Fabric

After a successful upload the file is visible in the Lakehouse explorer under:

```
Files/
└── raw/
    └── sample_raw_sales.csv
```

The bronze layer notebook (`01_ingest_raw_sales`) reads from this path.
